# GitHub 푸시 · Vercel 배포 가이드

이 폴더(`랜딩 수정1`)는 **랜딩페이지 제작** 프로젝트를 배포용으로 복사한 사본입니다.  
`node_modules`, `.next`, `.git`은 포함하지 않습니다. (클론/배포 후 `npm install` 필요)

---

## 1. 로컬에서 설치 및 빌드 확인

```powershell
cd "C:\Users\user\Desktop\보험광고 프로젝트\랜딩 수정1"
npm install
npm run build
```

빌드가 성공하면 Vercel 배포도 동일하게 통과할 가능성이 높습니다.

---

## 2. GitHub에 푸시

1. [GitHub](https://github.com/new)에서 새 저장소를 만듭니다. (예: `insurance-landing`)
2. 아래를 **저장소 루트**에서 실행합니다.

```powershell
cd "C:\Users\user\Desktop\보험광고 프로젝트\랜딩 수정1"
git init
git add .
git commit -m "Initial commit: landing page (Next.js)"
git branch -M main
git remote add origin https://github.com/<본인계정>/<저장소이름>.git
git push -u origin main
```

- SSH를 쓰는 경우: `git remote add origin git@github.com:<본인계정>/<저장소이름>.git`
- 이미 `git init`된 상태라면 `remote add`만 하거나 URL을 수정합니다.

---

## 3. Vercel에 배포

1. [vercel.com](https://vercel.com) 로그인 후 **Add New… → Project**
2. **Import**에서 위에서 만든 GitHub 저장소 선택
3. 설정 확인 (대부분 자동 인식):
   - **Framework Preset:** Next.js
   - **Root Directory:** `./` (프로젝트 루트)
   - **Build Command:** `npm run build` (기본값)
   - **Output Directory:** Next.js 기본 (비워 두거나 자동)
   - **Install Command:** `npm install` (기본값)
4. **Environment Variables:** 별도 API 키를 쓰지 않으면 추가 없이 **Deploy**

배포가 끝나면 `*.vercel.app` 주소로 사이트가 열립니다. 커스텀 도메인은 Vercel 프로젝트 **Settings → Domains**에서 연결합니다.

---

## 4. 포함된 주요 경로

| 내용 | 경로 |
|------|------|
| 앱 라우트 | `app/page.tsx`, `app/layout.tsx` |
| Hero 등 랜딩 블록 | `components/landing/` |
| 정적 자산 | `public/` (`public/images/hero-couple.jpg` 등) |

---

## 5. 주의사항

- `.env*.local`은 `.gitignore`에 있어 커밋되지 않습니다. Vercel에 필요한 환경 변수가 있으면 대시보드에서 직접 등록하세요.
- `pnpm`을 쓰려면 Vercel에서 Install Command를 `pnpm install`로 바꾸고 Lockfile을 맞추면 됩니다. (현재 폴더에는 `package-lock.json` 기준 **npm** 사용을 권장합니다.)
