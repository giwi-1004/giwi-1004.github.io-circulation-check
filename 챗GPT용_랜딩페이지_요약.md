# 랜딩페이지 요약 (ChatGPT / AI 공유용)

이 파일은 **코드를 실행하지 않고** 랜딩의 구조·문구·흐름을 AI에게 전달하기 위해 만든 **전용 스냅샷**입니다.  
챗GPT에는 **이 파일을 첨부**하거나 본문을 **복사해 붙여넣기** 하면 됩니다.

---

## 1. 프로젝트·기술

| 항목 | 내용 |
|------|------|
| 프레임워크 | Next.js 16 (App Router), React 19 |
| 스타일 | Tailwind CSS, 모바일 우선, 최대 콘텐츠 너비 약 720px |
| 폰트 | Noto Sans KR (`app/layout.tsx`) |
| 언어 | 페이지 `lang="ko"` |

---

## 2. SEO 메타데이터

- **title:** 내 보험 순환계 보장 확인 | 무료 점검  
- **description:** 내 보험에 순환계 보장이 빠져 있을 수 있습니다. 1분 안에 무료로 확인해보세요.

---

## 3. 페이지 구성 순서 (위→아래)

1. 히어로  
2. 자기 인식(체크 리스트)  
3. 순환계 중요성  
4. 비용 충격(치료비 카드 3개)  
5. 보험 연결 설명  
6. CTA(체크 유도)  
7. 체크 질문(토글 선택)  
8. 결과 영역 — **체크 완료 후에만** 표시  
9. 신청 폼(이름·전화·개인정보 동의)  
10. 카카오톡 1:1 문의(QR·링크)  
11. FAQ(아코디언)  
12. 푸터 고지  
13. 성공 모달 — **폼 제출 성공 시** 오버레이

## 4. 섹션별 문구·요소

### 4.1 히어로 (`HeroSection`)

- 보험은 다들 있습니다  
- 하지만  
- **뇌·심장 치료비는 빠져 있는 경우가 많습니다**  
- 내 보험에 있는지 1분 안에 확인해보세요  
- 이미지: `/images/hero-heart-pain.png` (심장 통증으로 가슴을 움켜쥔 남성)  
- 버튼: **빠진 보장 확인하기** → `#check-questions`로 스크롤  

### 4.2 혹시 이런 경험 (`SelfAwarenessSection`)

- 제목: **혹시 이런 경험 있으신가요?**  
- 항목(체크 아이콘 리스트):  
  - 최근 어지럼을 느낀 적이 있다  
  - 가슴이 답답한 느낌이 든 적 있다  
  - 가슴 통증을 경험한 적이 있다  
  - 심장이 두근거린 적이 있다  
  - 숨이 차는 경우가 종종 있다  
  - 보험 보장을 확인해본 적이 없다  

### 4.3 순환계 중요성 (`ImportanceSection`)

- 제목: **순환계 질환은 / 생각보다 흔하게 발생합니다**  
- 그리고 문제는 **치료비**입니다  
- 이미지: 컴포넌트 로컬 `importance-illustration.png` (두통으로 고통스러워하는 여성)  

### 4.4 비용 충격 (`CostShockSection`)

| 제목 | 강조 금액 문구 | 부가 설명 |
|------|----------------|-----------|
| 스텐트 삽입술 (PCI) | 최소 1,000만원 이상 발생 | 갑작스럽게 발생합니다 |
| 심장 수술 (개심술 등) | 최소 4,000만원 이상 발생 | 대부분 대비 없이 맞이합니다 |
| 뇌혈관 수술 | 최소 2,000만원 이상 발생 | 치료 이후 비용이 더 늘어납니다 |

### 4.5 보험 연결 (`InsuranceConnectionSection`)

- 문제는 / 기존 보험처럼 진단금·수술비만으로는  
- 실제 치료 과정에서 발생하는 비용을 / 충분히 대비하기 어렵다는 점입니다  
- 순환계 주요치료비 보장은 / **치료 내용에 따라 보험금이 지급되는 구조로**  
- 치료 단계에 맞춰 / 실질적인 보장을 받을 수 있습니다  

### 4.6 CTA 체크 유도 (`CtaCheckSection`)

- 1분이면, 상담 없이 확인 가능합니다  
- 버튼: **내 보험 체크하기** → `#check-questions` 스크롤  

### 4.7 체크 질문 (`CheckQuestionsSection`, `id="check-questions"`)

- 내 보험, / **제대로 준비되어 있을까요?**  
- 1분이면 확인 가능합니다  
- 아래 항목 중 해당되는 내용을 선택해주세요  
- 선택 항목(토글, 다중 선택):  
  1. 보험 가입 후 5년 이상 점검하지 않음  
  2. 심장·뇌 보장 여부를 정확히 모름  
  3. 가입한 특약 내용을 제대로 모름  
  4. 가족력 또는 건강 문제가 있어 걱정됨  
  5. 보험료 대비 보장이 적절한지 모름  
- **2개 이상 해당되면** / 보험 점검이 필요할 수 있습니다  
- 버튼: **결과 확인하기** → 선택 개수에 따라 결과 섹션 표시 및 `#result` 스크롤  

### 4.8 결과 (`ResultSection`, `id="result"`)

- **선택 2개 이상:**  
  - 제목: 순환계 보장 점검이 / 필요할 가능성이 있습니다  
  - 본문: 현재 보험 구조에 따라 / 보장이 부족할 수 있습니다  
- **선택 2개 미만:**  
  - 제목: 현재 상태는 양호하지만 / 점검을 권장합니다  
  - 본문: 정확한 보장 내용을 / 확인해보시는 것이 좋습니다  
- 버튼: **무료 확인하기** → `#application-form` 스크롤  

### 4.9 신청 폼 (`ApplicationFormSection`, `id="application-form"`)

- 제목: **무료 확인 요청**  
- 보험 가입 권유가 아닌 / 현재 보장 확인 목적입니다  
- 필드: 이름(placeholder: 이름을 입력해주세요), 전화번호(010-0000-0000)  
- **[필수] 개인정보 수집 및 이용에 동의합니다** + **[내용 보기]** (접기/펼치기, 스크롤 영역)  
- 개인정보 **전문 원문**은 저장소 `lib/privacy-consent-full-text.ts` (또는 `data/privacy-consent-full.txt`)에 있음 — 이 요약 파일에는 전체를 싣지 않음.  
- 제출 버튼: **보상 가능성 확인하기** (동의 전에는 비활성 등 UX는 코드 참고)  
- 성공 시: `SuccessModal`  

### 4.10 카카오톡 (`KakaoInquirySection`)

- 제목: **카카오톡 1:1 문의**  
- 전화보다 편하게 메시지 한 줄이면 가능합니다  
- 혜택 태그: 1:1 확인, 무료 검토, 빠른 답변  
- 아래 QR코드를 스캔하세요 — 이미지 `/kakao-qr.png`  
- 링크 버튼: **카카오로 내 상황 확인하기** (환경변수 `NEXT_PUBLIC_KAKAO_CHAT_URL` 없으면 기본 오픈채팅 URL 사용)  

### 4.11 FAQ (`FAQSection`)

| 질문 | 답변 요지 |
|------|-----------|
| 정말 무료인가요? | 보장 확인은 완전 무료. 가입 권유 없이 현재 보장 상태만 확인. |
| 어떤 서류가 필요한가요? | 보험증권·보험 앱 화면. 없으면 보험사 조회 도움. |
| 상담 시간은? | 전화 상담 기준 약 10~15분. |
| 가입 권유를 받게 되나요? | 아니요. 원하지 않으면 추가 연락 없음. |

### 4.12 푸터

- 본 페이지는 보험 상담 서비스를 안내하며, / 특정 보험 상품 가입을 권유하지 않습니다.  

### 4.13 성공 모달 (`SuccessModal`)

- **요청이 완료되었습니다**  
- 입력하신 번호로 / 빠른 시일 내 연락드리겠습니다  
- 버튼: **확인**
